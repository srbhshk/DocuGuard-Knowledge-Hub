# Contributing Guidelines

## Branching and Pull Requests
- Create feature branches from `main`.
- Keep pull requests focused and small.
- Ensure CI passes before merge.

## Development Workflow
1. Install backend and frontend dependencies.
2. Run lint/test locally before opening a PR.
3. Use pre-commit hooks to catch formatting/lint issues early.

## Commit Quality
- Write clear commit messages with intent and scope.
- Include documentation updates when behavior or setup changes.
